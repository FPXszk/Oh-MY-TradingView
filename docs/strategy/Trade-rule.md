# Trade Rule

## 目的

この文書は、銘柄の新規購入、追加購入、保有継続、縮小、撤退を判断するときの**売買ルール正本**です。

ユーザーが「この銘柄を買いたい」「この銘柄を持ち続けてよいか」とAIへ確認した場合、AIは最新情報、スクリーナー結果、チャート状態、イベントリスク、ポジションリスクを順番に確認し、最終的に `GO / STAY / STOP` のいずれかを返します。

詳細な定量化の調査背景は [dr-k-chart-strategy-quantification-report_20260707.md](dr-k-chart-strategy-quantification-report_20260707.md) に残します。ただし、実際の売買判断ではこの `Trade-rule.md` を優先し、調査レポートの仮条件を自動的にルール扱いしません。

## 対象スタイル

- AI・半導体関連を中心とする大型株、テーマ内リーダー株のスイング
- 現物または信用取引
- 原則2〜5銘柄へ絞る集中運用
- 明確なセットアップがない銘柄は取引しない
- 弱い銘柄の逆張りやナンピンではなく、強い銘柄の低リスクな買い位置を狙う

## 1. 判定の意味

| 判定 | 新規購入・追加購入 | 既存ポジション |
|---|---|---|
| `GO` | 現在の価格帯でエントリーまたは追加を許可できる | 条件を満たす場合は保有継続または計画内の追加が可能 |
| `STAY` | 今は買わず、監視またはセットアップ完成を待つ | 保有は継続できるが、追加は行わない |
| `STOP` | 新規購入しない | 縮小、利確、損切り、撤退のいずれかを優先する |

`STAY` は「何となく様子を見る」ではありません。何を待っているかを、価格、ピボット、決算日、スクリーナー更新、チャート状態などで明示します。

情報不足、最新データ未確認、判定不能は `GO` にしません。原則 `STAY` とし、重大なリスクを確認した場合は `STOP` とします。

## 2. 売買前の必須確認順序

AIは次の順番を省略せず確認します。

1. **依頼内容の確定**
   - 銘柄コード、市場、新規購入・追加購入・保有判断のどれか
   - 想定エントリー価格、現在の保有数量と平均取得価格
   - 自己資金、現在の建玉総額、同一テーマの保有状況
2. **最新ニュースと世の中の情勢**
   - 会社固有ニュース、決算、ガイダンス、増資、規制、訴訟、製品事故
   - 金利、為替、政策、地政学、指数急変など、その銘柄へ影響する市場材料
   - 情報の発生日と確認時刻を明示する
3. **市場の地合い**
   - 主要指数のトレンド
   - 上昇銘柄と下落銘柄の広がり
   - 新高値・新安値、直近ブレイクの成功率
   - `RISK_ON / NEUTRAL / RISK_OFF` のどれかに分類する
4. **セクター・業種の強さ**
   - 市場全体に対する相対強度
   - セクター内の上昇銘柄比率と新高値銘柄
   - 上位リーダー株が崩れていないか
5. **最新スクリーナー結果**
   - 最新成功runの日時
   - 業種・インダストリー順位
   - 個別銘柄順位、スコア、採用理由、除外理由
   - 対象銘柄が現在の上位候補へ入っているか
6. **リーダー株判定**
   - セクター内相対強度
   - 52週高値からの距離
   - 短期移動平均線上の滞在率
   - 上昇日の出来高と下落日の出来高
   - 同業他社より先に上昇しているか
7. **チャート状態と買い位置**
   - 有効なセットアップ、ピボット、エントリー範囲
   - ピボットからの乖離率
   - 損切り価格、想定損失、リスクリワード
8. **ポートフォリオへの影響**
   - 追加後の建玉総額
   - 同一テーマ・同一値動きの重複
   - すべての損切りが発動した場合の想定損失
9. **最終判定**
   - `GO / STAY / STOP`
   - 判定を変える条件
   - 実行する場合のエントリー、数量、損切り、初期目標

## 3. 最新情報の扱い

- 株価、ニュース、決算予定、スクリーナーは、判断時点で取得できる最新情報を使う
- スクリーナーは最新の正常完了結果を使い、古い結果しかない場合は古い日付を明記する
- 最新の取引日を反映していないスクリーナーしか確認できない場合、新規判断は原則 `STAY`
- ニュースの見出しだけで判断せず、発生日、一次情報、会社発表、規制当局資料を優先する
- SNSや投稿者の意見だけを根拠に `GO` を出さない
- データが取得できなかった項目は「問題なし」と推定せず、未確認として表示する

## 4. スクリーナーの必須条件

- 対象銘柄が最新スクリーナーの上位候補へ入っているかを必ず確認する
- 上位候補へ入っていない銘柄は、チャートだけを理由に `GO` としない
- ランク外でも監視価値がある場合は `STAY` とし、ランク入りまたはスコア改善を待つ
- 業種が上位でも、個別銘柄が弱い場合は買わない
- 個別銘柄が上位でも、業種全体と主要リーダーが崩れている場合は買わない
- スクリーナー順位だけで売買せず、チャート位置とリスク条件を最後に確認する

AIは最低限、次を出力します。

```text
screener_run:
industry_rank:
stock_rank:
score:
selection_reasons:
exclusion_reasons:
```

## 5. 市場・セクター・リーダー株のゲート

### 市場

次のような状態では `GO` を出しません。

- 主要指数が短期・中期移動平均線を明確に割り、傾きも下向き
- 下落銘柄、新安値銘柄、失敗ブレイクが急増
- 直近のブレイク成功率が明確に低下
- 悪材料へ市場全体が過敏に反応している

`RISK_OFF` では新規購入を原則停止します。既存ポジションは、個別チャートが崩れる前でもサイズ縮小を検討します。

### セクター・業種

- セクターの相対強度が市場平均より強いこと
- 一部の1銘柄だけでなく、複数銘柄へ資金が広がっていること
- 上位リーダー株が同時に崩れていないこと
- セクター内の上昇銘柄比率、新高値比率、出来高流入を確認すること

### リーダー株

- 強い領域の出遅れ銘柄ではなく、先導している上位1〜2銘柄を優先する
- 52週高値圏にあり、上昇トレンドを維持していること
- 5日線・10日線上の滞在率が高く、大陰線や高値からの急落が少ないこと
- 同じサブグループの銘柄を複数持ち、実質的に同じポジションを重ねない

## 6. チャート状態

銘柄は、単純な点数だけではなく現在の状態で判断します。

| 状態 | 意味 | 新規判断 |
|---|---|---|
| `BASE_FORMING` | 持ち合い形成中 | `STAY` |
| `BREAKOUT_READY` | ピボット直下で値幅が締まっている | 他の条件を満たせば `GO` 候補 |
| `BREAKOUT_TRIGGERED` | 当日または直近で有効なブレイク | 乖離と出来高を確認して `GO` 候補 |
| `EXTENDED` | 有効ピボットから上へ離れすぎ | `STAY`。飛びつかない |
| `PULLBACK_FORMING` | ブレイク後の押し目形成中 | `STAY` |
| `NU_READY` | N字・U字の新しいピボット形成中 | 上抜け確認までは `STAY` |
| `ABSORPTION` | 高値付近で売り圧を吸収中 | 上抜け確認までは `STAY` |
| `TREND_HOLD` | 初玉を保有しトレンドを追う状態 | 既存保有は `STAY`、無条件追加はしない |
| `FAILED_BREAKOUT` | ブレイク後にピボット下へ失速 | `STOP` |
| `SETUP_BROKEN` | 支持線、直近安値、重要MAを明確に割った | `STOP` |
| `NO_TRADE` | 地合い、位置、流動性、形のいずれかが不適切 | `STOP` |

### 許可する主な買い位置

- 数週間から数か月のベースを抜く初動ブレイク
- 高値圏で形成した小さなN字・U字の新しいピボットブレイク
- 高値付近で値幅と出来高が縮小した吸収型セットアップの上抜け
- 強い上昇トレンド中に、5日線・10日線・25日線へ引き付けた押し目からの再上昇

単に移動平均線へ到達しただけでは買いません。反発、再上昇、または新しいピボット突破を確認します。

## 7. ピボットと飛びつき禁止

```text
extension_pct =
  (current_price - active_pivot_price)
  / active_pivot_price
  × 100
```

- 有効なピボットから**5%以上上**にあり、新しいセットアップが形成されていない場合は `EXTENDED`
- `EXTENDED` では新規購入と追加購入を行わない
- 上昇後に新しいN字、U字、三角持ち合い、吸収型セットアップを作った場合、その新しい上限を `active_pivot_price` として再評価する
- ピボットが特定できない場合は `GO` を出さない
- ブレイク後に追いかけるのではなく、5日線・10日線の接近または新しいセットアップを待つ

## 8. エントリー条件

`GO` 候補とするには、最低限すべてを満たします。

- 最新情報と最新スクリーナーを確認済み
- 市場が `RISK_OFF` ではない
- セクター・業種が弱くない
- 対象がテーマまたは業種内のリーダー株
- `BREAKOUT_READY` または有効な `BREAKOUT_TRIGGERED`
- 最新ピボットから5%以内
- 明確な損切り価格を事前に決められる
- 初期リスクリワードが最低2以上、理想は3以上
- 決算などのイベントリスクを確認済み
- ポジションサイズと総建玉が上限内
- 同一テーマの重複保有にならない

出来高は、ブレイクの信頼性を上げる確認材料として使います。通常のブレイクでは直近平均より明確に増えていることを優先し、異常な出来高増加は特別な強さとして別に表示します。

## 9. 損切りルール

損切りは、固定率より先にチャート構造から決めます。

優先順位は次のとおりです。

1. ピボット下
2. セットアップ内の直近安値下
3. 支持線または重要移動平均線を割り、セットアップ崩れが確定する位置
4. 上記が使えない場合のみ、買値から7〜8%を最大許容幅として使う

- 構造上必要な損切り幅が8%を超える場合は、ポジションを小さくするのではなく原則そのエントリーを見送る
- 損切り価格を決められない場合は `STOP`
- ストップ到達後に理由を付けて保有継続しない
- ナンピンによって平均取得価格を下げ、当初の損切りを無効化しない
- 窓開け急落では逆指値価格より不利に約定する可能性を前提に、決算・重要イベント前のサイズを管理する

## 10. ポジションサイズ

### 1トレードの最大損失

平時は、1回のトレードで許容する最大損失を**自己資金の2%以内**とします。

```text
allowable_loss = account_equity × 0.02
risk_per_share = entry_price - stop_price
shares_by_risk = floor(allowable_loss / risk_per_share)
```

実際の数量は、次のうち最も小さい数量です。

```text
actual_shares = min(
  shares_by_risk,
  shares_by_position_cap,
  shares_by_buying_power
)
```

### 1銘柄あたりの上限

- 通常は自己資金の25%程度を基準にする
- 最大でも自己資金の50%を超えない
- 50%まで保有できるのは、損切り幅が狭く、最大損失2%以内へ収まる場合だけ
- 例: 25%ポジション × 8%損切り = 自己資金の2%損失
- 例: 50%ポジション × 8%損切り = 自己資金の4%損失となるため禁止
- 50%ポジションなら、2%リスクへ収めるため損切り幅は最大4%が目安

### ポートフォリオ全体

- 2〜5銘柄へ絞る
- すべてのストップが発動した場合の想定損失合計を毎回計算する
- 平時の想定損失合計は自己資金の6%以内を上限とする
- 同じセクターやサブグループの重複は、銘柄数ではなく1つのリスクとして扱う
- 上限を超える場合は、既存ポジションを縮小するか新規購入を `STOP` とする

## 11. レバレッジと信用取引

- 平時の最大建玉総額は自己資金の1.25〜1.5倍
- 直近トレードが機能し、地合いが強い上昇トレンドにある場合だけ最大2倍まで許可する
- 損失が出始めた時、失敗ブレイクが増えた時、地合いが崩れた時は1倍以下へ落とす
- レバレッジは勝っている局面でだけ使う
- 信用金利は必要経費として扱えるが、短期の入れ替え余地を失うほど信用枠を固定しない
- 信用フルポジションで決算や重要イベントを跨がない

## 12. 利確と保有管理

- 買値から20〜30%上昇したら、そのトレードを損で終わらせない管理へ切り替える
- 原則としてストップを建値付近まで引き上げる
- その後は10日線、25日線、直近安値、直近の新しいピボットを使ってトレイリングする
- 強い銘柄が5日線に沿って上昇している間は、短期モメンタムが維持されているとみなす
- 回転玉は10日線割れを部分利確の候補にする
- 長期間5日線・10日線を割らず上昇した銘柄が25日線まで下落した場合は、同じ短期トレンドが続いていると決めつけない
- 含み益が大きい初玉と、後から追加した回転玉を分けて管理する
- 含み益が乗った後も、セクターリーダー全体の崩れと市場の悪化を無視しない

## 13. 決算・イベント跨ぎ

- 含み益が十分にない状態で信用フルポジションのまま決算を跨がない
- 決算前に10〜15%以上の含み益がない場合、半分から全部を利確または縮小する
- クッションが不足している場合は、決算後の値動きを確認してから入り直す
- 新規購入前に次回決算日を確認できない場合は `STAY`
- 決算、FDA判断、裁判、規制判断、重要製品発表など、価格が飛ぶ可能性のあるイベントを通常のチャートリスクと同一扱いしない

## 14. ハード停止条件

次のいずれかに該当する場合、点数や期待が高くても `GO` を出しません。

- 最新ニュース、価格、決算予定、スクリーナーを確認できない
- 市場が `RISK_OFF`
- セクターと主要リーダーが同時に崩れている
- 対象銘柄が最新スクリーナー上位候補へ入っていない
- `FAILED_BREAKOUT`、`SETUP_BROKEN`、`NO_TRADE`
- 有効ピボットから5%以上上で、新しいセットアップがない
- ピボットまたは損切り価格を決定できない
- 必要な損切り幅が8%を超える
- 初期リスクリワードが2未満
- 流動性が不足している
- セットアップの値幅と出来高が拡大し、締まりがなくなっている
- 追加後の1トレード損失またはポートフォリオ想定損失が上限を超える
- 同じ値動きの銘柄を重複保有する
- 決算などのギャップリスクを許容できない

## 15. 判定基準

### `GO`

- 必須情報をすべて確認済み
- スクリーナー上位候補
- 市場・セクター・リーダー条件を満たす
- チャート状態が `BREAKOUT_READY` または有効な `BREAKOUT_TRIGGERED`
- ピボットから5%以内
- 損切りが明確で8%以内
- リスクリワード2以上
- 資金管理とイベントリスクの条件を満たす

### `STAY`

- `BASE_FORMING`、`PULLBACK_FORMING`、`NU_READY`、`ABSORPTION`
- `EXTENDED` だが、トレンド自体は維持している
- スクリーナー更新待ち、ランク入り待ち
- 決算通過待ち
- 地合い、セクター、チャートのどれかが中立
- 必要データの一部が未確認
- 既存保有は維持できるが追加条件を満たさない

### `STOP`

- ハード停止条件へ該当
- 投資仮説を壊す会社固有材料が発生
- `FAILED_BREAKOUT`、`SETUP_BROKEN`、`NO_TRADE`
- 損切り、縮小、撤退を優先すべき状態

## 17. 運用の優先順位

1. 最優先は退場しないこと
2. ニュース、地合い、スクリーナー、チャートの順序を飛ばさないこと
3. 強いセクターのリーダー株だけを対象にすること
4. セットアップが完成した低リスク位置だけで入ること
5. 利益は伸ばし、損失は初動で切ること
6. 条件が揃わない時は現金のまま待つこと

## 要点

- `Trade-rule.md` が売買判断の正本
- 最新情報、スクリーナー、地合い、セクター、リーダー、チャート、資金管理を一体で判断する
- ランク上位でも買い位置が悪ければ `STAY`
- チャートが良くてもランク外、地合い不良、リスク過大なら `GO` にしない
- ピボットから5%以上離れた銘柄へ飛びつかない
- 1銘柄の大きさではなく、損切り時の実損失から数量を決める
- 判定不能を楽観的に処理しない
