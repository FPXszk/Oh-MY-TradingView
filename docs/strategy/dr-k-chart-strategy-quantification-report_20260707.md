# Dr.K チャート投資戦略の定量化・自動判定 調査レポート

## 結論

**実現性はかなり高いです。**

Dr.K氏の判断は一見「チャートを見た感覚」に見えますが、投稿を分解すると、実際には次の順番で判断しています。

> **地合い → 強いセクター → リーダー株 → 相場の段階 → セットアップの締まり → ブレイク位置 → 損切り位置**

したがって、単純な画像認識AIよりも、

1. OHLCVから形を数値化
2. 銘柄を「現在の状態」に分類
3. 状態ごとにスコアリング
4. 最後にAI画像判定を補助利用

という構成が適しています。

今回、公開検索で確認できた**2024年12月～2026年7月の投稿**を中心に、チャート・エントリー・押し目・損切り・地合いに関する発言を重複統合しました。ただし、Xの公開検索に表示されない投稿もあるため、「全投稿を完全取得したアーカイブ」ではありません。完全網羅にはX APIや本人の投稿エクスポートが必要です。

# 1. Dr.K氏のチャート戦略を言語化した全体像

## 相場初期：持ち合いブレイクで初玉を作る

最低でも数か月続いたレンジでエネルギーが蓄積され、突然大幅な出来高増加を伴って抜ける銘柄を探しています。「10倍以上の出来高」は、通常のブレイクすべての必須条件というより、特に大化け候補を見つけるための強条件と考えるのが自然です。([x.com](https://x.com/Drdebuneko/status/2051171801158586698))

本人が買うトリガーとして明示しているのは、主に次の2つです。

- 明確な価格節目のブレイク
- トレンドラインのブレイク

イビデンの例では、同じ上昇トレンド内で複数回入り直し、10日線割れで部分利確する運用も説明しています。([x.com](https://x.com/Drdebuneko/status/2068321904189169886))

## 相場中盤：高値のN字・U字で回転する

理想形は、下の持ち合いブレイクで取得した初玉をできるだけ保有しつつ、上昇中盤以降に高値圏で形成されるN字・U字を使って追加・回転する形です。([x.com](https://x.com/Drdebuneko/status/2073384226502943011))

ブレイクを見逃した場合も、そのまま飛びつくのではなく、

- 5日線・10日線が追いつく
- 小さなN字・U字が形成される
- その新しい小型セットアップを抜く

という順番を待つことを勧めています。([x.com](https://x.com/Drdebuneko/status/2052896276904387065))

## 高値で一度止められても、すぐには捨てない

N字で前回高値を一度で超えられなくても、高値付近で横ばいを続けている場合は、売り圧力を吸収している可能性があり、その後に上昇することがあると説明しています。([x.com](https://x.com/Drdebuneko/status/2067142178728714399))

つまり、単なる「高値を抜けなかった失敗」ではなく、

- 高値から大きく離れない
- 安値を切り下げない
- 値幅が縮小する
- 高値付近に滞在する

なら、**吸収型セットアップ**として残す考え方です。

## ブレイクから離れすぎた場所では買わない

本人が数値としてかなり明確に述べているのが、

> ブレイク地点から5％以上上で買うと、損切りになる確率が高い

という経験則です。([x.com](https://x.com/Drdebuneko/status/2067810666560508044))

ただし、これは「最初のブレイク地点から永遠に5％以内」という意味ではありません。

株価が上昇したあとに新しいN字・U字・三角持ち合いを作った場合は、**その新しいパターンの上限が次のピボット**になります。したがってプログラムでは、常に「最新の有効なセットアップのピボット」からの乖離率を計算する必要があります。

## 移動平均線の役割

投稿から読み取れる役割分担は次のとおりです。

| 移動平均線 | 主な意味 |
|---|---|
| 5日線 | 非常に強いモメンタムの維持、短期の押し目 |
| 10日線 | 回転玉の管理、部分利確や入り直し判断 |
| 25日線 | 日本株の上昇トレンド継続を測る中期基準 |
| 週足・月足 | 上値抵抗、相場の若さ、大きな値幅余地の確認 |

強い銘柄は5日線からほとんど離れず上昇することがあり、5日線への沿い方そのものを強さの判断材料にしています。([x.com](https://x.com/Drdebuneko/status/1871793350145392999))

一方、長く5日線や10日線を割らずに上昇してきた銘柄が25日線まで下がる場合は、それまでと同じ短期トレンドではなく、変化が発生した可能性を考える必要があります。([x.com](https://x.com/Drdebuneko/status/2068118843999010853))

2倍・3倍の大きな上昇を狙う場合は、日足だけでなく、週足・月足のセットアップと過去の抵抗価格帯を見る必要があるとも述べています。([x.com](https://x.com/Drdebuneko/status/2058530729546723821))

# 2. 投稿から抽出できた機械化可能なルール

| 本人の判断 | プログラム上の解釈 |
|---|---|
| 数か月のレンジから出来高急増で抜ける | 長期ベース＋出来高ブレイク |
| 節目かトレンドライン抜けしか買わない | 明確なピボットが存在しない銘柄を除外 |
| 三角持ち合い抜けが買い位置 | 高値切り下げ＋安値切り上げの収縮検出 |
| ブレイク地点から5％以上を追わない | `extension_pct > 5%`なら元のセットアップでは新規禁止 |
| 5日線・10日線を引き付ける | 株価と短期MAの乖離縮小を待つ |
| 小さなN字・U字を抜いて買う | 押し目後の新規ピボットブレイク |
| 高値付近の横ばいを捨てない | 高値滞在＋値幅縮小＝売り圧吸収 |
| 5日線に沿う株は強い | MA5上の滞在率・MA5割れ回数を評価 |
| 10日線割れで部分利確 | 短期回転玉のトレーリング条件 |
| 25日線は上昇継続の目安 | 中期トレンド判定 |
| 地合いが悪いとブレイク成功率が落ちる | 市場レジームを事前ゲートにする |
| 所属セクター全体が上昇しているかを見る | セクター騰落率・ブレッドス・相対強度 |
| リーダー株の崩れを危険信号にする | セクター上位株のセットアップ崩壊率 |
| 上昇トレンド後期は失敗が増える | 相場段階に応じてブレイクスコアを減点 |
| 甘く緩いセットアップを買わない | ベース幅・ATR・下ヒゲ・ギャップの乱れを減点 |
| セットアップがなければ触らない | 判定不能を無理にランキングしない |

地合いと所属セクターを最重要条件に挙げており、地合いが悪いとブレイクアウト確率が著しく低下すると説明しています。([x.com](https://x.com/Drdebuneko/status/2034841303566229988))

また、上昇トレンド後期では、見た目上セットアップが整っていてもブレイク失敗が増えると述べています。([x.com](https://x.com/Drdebuneko/status/2072640877621834063))

リーダー株のセットアップが崩れた局面では、フルポジションを避け、ポジションを軽くする姿勢も一貫しています。([x.com](https://x.com/Drdebuneko/status/2070476815672746449))

# 3. 一番重要な設計：点数より先に「状態」を判定する

この戦略を一つの0～100点だけで表すと、かなり情報が失われます。

まず銘柄を以下の状態に分類すべきです。

| 状態 | 意味 | システム表示 |
|---|---|---|
| `BASE_FORMING` | 持ち合い形成中 | 監視 |
| `BREAKOUT_READY` | ピボット直下で締まっている | 今すぐ触れる候補 |
| `BREAKOUT_TRIGGERED` | 当日または直近でブレイク | エントリー候補 |
| `EXTENDED` | ピボットから離れすぎ | 飛びつき禁止 |
| `PULLBACK_FORMING` | ブレイク後の押し目形成中 | 待機 |
| `NU_READY` | 小さなN字・U字が完成直前 | 再エントリー候補 |
| `ABSORPTION` | 高値付近で売り圧吸収中 | 継続監視 |
| `TREND_HOLD` | 初玉を握る状態 | 保有継続 |
| `FAILED_BREAKOUT` | ピボットを抜いた後に失速 | 撤退候補 |
| `SETUP_BROKEN` | 支持線・MA・直近安値割れ | 損切り |
| `NO_TRADE` | 緩い・地合い不良・位置不良 | 見送り |

ユーザーが求めている分類は、この状態モデルでほぼ表現できます。

- **今すぐ触れるもの**  
  `BREAKOUT_READY` / `BREAKOUT_TRIGGERED`

- **ブレイク後に緩い押し目が来ているもの**  
  `PULLBACK_FORMING`

- **上抜けを見逃したが後から入りそうなもの**  
  `NU_READY` / `ABSORPTION`

- **すでに上がりすぎているもの**  
  `EXTENDED`

# 4. 定量化仕様案 v0.1

以下の数値は、本人が明示した「5％」「5・10・25日線」「数か月」「10倍出来高」以外は、まずバックテストを開始するための**仮の初期値**です。

## 共通指標

```text
ATR_PCT       = ATR(20) / Close
REL_VOLUME    = Volume / MedianVolume(20)
PIVOT_DIST    = (Close - PivotPrice) / PivotPrice
PIVOT_DIST_ATR= (Close - PivotPrice) / ATR(20)

RS_20         = StockReturn(20) - BenchmarkReturn(20)
SECTOR_RS_20  = SectorReturn(20) - BenchmarkReturn(20)
```

ATRで正規化することで、値動きの大きい小型株と大型株を同じ基準で比較しやすくなります。

## 4-1. ベース形成

```text
base_days             >= 40営業日
base_width_pct         <= 25%
last_20d_width
    < first_20d_width
ATR20 / ATR60          < 0.8
down_volume / up_volume が縮小
```

### 締まりスコア

```text
tightness_score =
    値幅収縮
  + ATR収縮
  + 出来高枯れ
  + 安値切り上げ
  - 大陰線回数
  - ギャップ乱発
  - 長い下ヒゲの乱発
```

「数か月」を厳格にするモンスターストック用では、60～120営業日程度まで条件を引き上げます。

## 4-2. ピボット価格

単純な過去最高値だけではなく、複数回止められた価格帯を検出します。

```text
pivot_price =
    過去高値クラスタの中央値
```

候補アルゴリズムは、

- ZigZagによるスイング高値抽出
- 高値価格をATR単位でクラスタリング
- 接触回数が2回以上ある抵抗帯を採用
- 最も直近かつ明確な抵抗帯をピボットに設定

です。

## 4-3. 「今すぐ触れる」ブレイク直前

```text
-1.5% <= PIVOT_DIST <= +2.0%
tightness_score >= 70
Close > MA20 > MA50
MA20_slope > 0
sector_score >= 60
market_regime != RISK_OFF
```

当日ブレイク成立は、

```text
Close > PivotPrice + 0.1 × ATR20
REL_VOLUME >= 1.5
ClosePositionInBar >= 0.7
```

などから開始します。

ここでの出来高1.5倍は一般的なブレイク検出用の仮値です。Dr.K氏が言及した10倍は、別途、

```text
monster_volume_flag = REL_VOLUME >= 10
```

として、最高ランクの特殊フラグにするのがよいです。

## 4-4. 飛びつき判定

```text
extension_pct =
    (CurrentPrice - ActivePivotPrice)
    / ActivePivotPrice
```

```text
extension_pct > 5%
AND 新しいセットアップが未形成
→ EXTENDED
```

重要なのは、N字やU字を形成したら、`ActivePivotPrice`を新しい小型ベースの上限に更新することです。

## 4-5. ブレイク後の押し目

```text
previous_breakout = true
trend_structure   = higher_high AND higher_low
pullback_depth    = 3～12%
pullback_atr      = 1～3 ATR
pullback_volume   < breakout_volume
CloseがMA5・MA10・MA25のいずれかに接近
```

良い押し目ほど、

- 下落時の出来高が減る
- 陰線の実体が小さくなる
- MA付近で下ヒゲまたは陽線が出る
- 元のピボットを割らない
- セクターの相対強度が落ちていない

という特徴を持つと仮定します。

## 4-6. N字判定

ZigZagで以下の4点を取ります。

```text
L0 → H1 → L1 → H2
```

条件例：

```text
H2 ≈ H1
L1 > L0
L1 > 元のブレイクピボット
H1からL1までの下落が1～3 ATR
H2突破でトリガー
```

```text
abs(H2 - H1) <= 0.5 ATR
```

なら、前回高値と同一の抵抗帯を試していると判定できます。

## 4-7. U字判定

N字より滑らかで長い調整です。

```text
前半：傾きがマイナス
中央：傾きがほぼゼロ
後半：傾きがプラス
右側安値 >= 左側安値
出来高は底付近で減少
```

実装候補は、

- 移動平均した価格への二次曲線回帰
- 局所傾きの符号変化
- Dynamic Time Warping
- Matrix Profileによる類似形状検索

です。

チャートパターンの自動認識自体は新しい考え方ではなく、Lo、Mamaysky、Wangはカーネル回帰を使ってチャート形状を体系的に認識する方法を提示しています。変化点検出にはPELT、過去の類似パターン探索にはMatrix Profileのような時系列手法が利用できます。([papers.ssrn.com](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=228099))

## 4-8. 高値での売り圧吸収

```text
Close >= H1 - 0.5 ATR の日数 >= 3
高値圏滞在日数            <= 15
ATR5 < ATR20
安値が切り下がっていない
大陰線がない
出来高が徐々に減少
```

この状態を、

```text
ABSORPTION
```

とします。

その後、

```text
Close > H1
AND REL_VOLUME上昇
```

で`NU_READY → BREAKOUT_TRIGGERED`へ遷移させます。

# 5. 地合い・セクター・リーダー株の定量化

この部分を省くと、形だけきれいな「失敗ブレイク」を大量に拾う可能性があります。

## 市場スコア

```text
market_score =
    指数がMA20・MA50より上
  + 指数MA20・MA50の傾き
  + 上昇銘柄数 / 下落銘柄数
  + 新高値銘柄数 / 新安値銘柄数
  + ブレイク成功率の直近推移
```

特に有効そうなのが、**システム自身が検出した直近のブレイク成功率**です。

```text
breakout_success_rate_20d =
    過去20日間のブレイクで
    +2Rへ先に到達した割合
```

これが急低下していれば、機械的に`RISK_OFF`へ寄せられます。

## セクタースコア

```text
sector_score =
    セクター20日相対リターン
  + セクター内MA20上銘柄比率
  + セクター内新高値銘柄比率
  + 上位3銘柄の平均RS
  + 出来高流入
```

## リーダー株スコア

```text
leader_score =
    セクター内RS順位
  + 52週高値までの距離
  + MA5・MA10上の滞在率
  + 上昇日の出来高
  + 決算後の値動き
  - 大陰線
  - 高値からの急落
```

リーダー株がまとめて崩れた場合は、

```text
sector_leader_breakdown_ratio >= 50%
→ sector_risk_off = true
```

のようなゲートを設けます。

# 8. AI画像判定はどこで使うべきか

AIを最初から主判定にするのではなく、定量ロジックの補助にします。

## AIに任せる部分

- N字とU字の境界が曖昧なケース
- 「締まって見える」「汚く見える」の補助判定
- 上値のしこりや過去の抵抗帯
- 多時間軸をまとめた最終評価
- 数値判定と見た目が食い違うケースの再確認

## AIに任せない部分

- ピボット価格
- 損切り価格
- 5％乖離
- ATR
- 出来高倍率
- MAとの距離
- リスクリワード
- 市場・セクター強度

理想は、

```text
ルール判定：80点
画像AI判定：強気
→ 最終候補

ルール判定：80点
画像AI判定：形が乱れている
→ 人間確認

ルール判定：45点
画像AI判定：強気
→ 原則見送り
```

という使い方です。

# 9. 学習・バックテスト方法

「翌日上がったか」だけでは、Dr.K氏の損小利大型戦略を正しく評価できません。

各シグナルについて、

```text
Entry
Stop = -1R
Target1 = +2R
Target2 = +3R
```

を設定し、

- 先に-1Rへ到達したか
- 先に+2Rへ到達したか
- 先に+3Rへ到達したか
- 最大含み益＝MFE
- 最大含み損＝MAE
- 到達までの日数
- 5、10、20、40営業日後のリターン

を保存します。

### 成功ラベル例

```text
SUCCESS_2R =
20営業日以内に
-1Rより先に+2Rへ到達
```

### 必須の検証方式

- 時系列順のウォークフォワード
- 学習期間とテスト期間を分離
- 同じ相場局面をランダム分割しない
- 分割・株式併合を調整
- 上場廃止銘柄を除外しない
- 同一セクター・時価総額帯との比較
- コスト、スリッページ、ストップ安を反映

Dr.K氏の投稿チャートは、パターンを言語化するための**教師データ候補**としては有効ですが、勝ちチャートだけが投稿されている可能性があるため、成績検証用の正解データには直接使わない方が安全です。

# 10. Oh-MY-TradingViewへの組み込み方

現在のスクリーニングの後段に、独立した評価モジュールとして追加するのがよいです。

```text
Phase 1～4
既存の市場・業種・個別銘柄スクリーニング
        ↓
Phase 5
Chart Setup Evaluation
        ↓
状態別ランキング
```

想定モジュール構成は次の形です。

```text
src/chart-setup/
├── market-regime.js

├── absorption-detector.js
├── risk-reward.js

```

処理順序は、

```text

10. 状態内ランキング
```

が最も自然です。

## 最終評価

この構想は、単なる「AIにチャート画像を見せて上がりそうか聞く」ものより、はるかに再現性の高いシステムにできます。

特に重要なポイントは次の3つです。

- **形だけでなく、地合い・セクター・相場段階まで数値化する**
- **一つの点数ではなく、現在の状態を分類する**
- **ブレイクを見逃した銘柄は、新しいN字・U字ピボットができた時だけ再評価する**

実装の最初の完成形は、**日足OHLCVを入力すると、状態・ピボット・エントリーゾーン・損切り・スコア・理由コードを返す「Chart Setup Evaluator v0.1」**にするのが最短です。
