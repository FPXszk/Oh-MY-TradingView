# SECURITY — セキュリティ制約・禁止操作

目的
- ソフトウェアの安全性を保ち、機密情報の漏洩や脆弱性を未然に防ぐための規約を定義する。

機密情報とシークレット管理
- 機密情報は環境変数で管理し、コードやドキュメントにハードコードしてはいけない。
- Git に秘密情報をコミットしない。万一コミットした場合は速やかに鍵をローテートし、履歴の除去を検討する。

ログの扱い
- ログに機密情報（トークン、パスワード、個人情報）を出力しない。出力する場合は適切にマスク/匿名化する。

アクセス制御
- 最小権限の原則に従い、サービスやユーザには必要最小限の権限を付与する。

レビューと承認
- セキュリティに影響する変更は必ずコードレビューで明示し、必要に応じてセキュリティレビューを行う。
