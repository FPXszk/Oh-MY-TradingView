# Windows Native Workspace Migration 20260620_1027

## Goal

WSL 側の既存リポジトリ `/home/fpxszk/code/Oh-MY-TradingView` を、Windows native 側の `C:\00_mycode\Oh-MY-TradingView` へ移行し、今後のローカル開発・Codex App 操作・Git push の正規作業場所を Windows 側にする。

## Assumptions

- WSL 側リポジトリは削除せず、バックアップ扱いで残す。
- `.git` はコピー対象に含め、履歴・branch・remote 設定を引き継ぐ。
- `node_modules`, `.next`, `dist`, `coverage`, `.cache` は再生成可能な重い生成物としてコピー対象から除外する。
- 現在の WSL 側 `origin` は SSH remote の `git@github.com:FPXszk/Oh-MY-TradingView.git` であり、GitHub 操作は既存設定を保存する。
- Windows PowerShell 実行は WSL から `powershell.exe` を起動して確認する。WSL interop が利用不能な場合は、その事実を記録し、Windows 側ファイルシステム上のコピー確認までを実施する。
