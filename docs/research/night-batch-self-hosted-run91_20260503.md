## 改善点と次回バックテスト確認事項

1. **tp パラメータ無効の原因調査**
   tp=8〜15 が全て同一結果 (avg_np=17,929.59, PF=8.7423) は、Pine Script 側で tp が正しく適用されていない可能性がある。次回実行前に `emr-ae-v13` の Pine Script を確認し、tp パラメータがトレードロジックに正しく組み込まれているか検証する。「tp=8 vs tp=15 で同一トレード数・同一損益」が再現するなら実装バグを疑う。

2. **tp=18 特異性の解明**
   tp=18 の q=20 のみが unique な結果（avg_np=18,956, 5/8成功）を示した。この設定で BTCUSD・QQQ・SPY が失敗した原因（タイムアウト？ポジションサイズ不足？）を調査し、8/8 完走できる条件を確認する。

3. **MSTR 集中リスクへの対処**
   上位全戦略で MSTR が net_profit の 48〜51% を占める。MSTR を除いた 7 シンボルでの avg_net_profit がどう変化するか再集計し、戦略の MSTR 依存度を定量評価する。MSTR を除外した場合の PF・DD を確認すること。

4. **q=0 の win_rate 19% 異常の理解**
   q=0 は win_rate が 19.03% と q=5 以上（38%台）に対して著しく低い。これが意図した動作（ポジション保有方法の違い）なのかバグなのかを確認する。q=0 が「クオリティ閾値なし」の意味なら挙動差は理解できるが、PF が高いまま win_rate が低い理由を文書化する。

5. **tp=25 失敗原因の特定**
   tp=25 の全 9 戦略が 0/5 success。取引が成立しない（利確条件が厳しすぎてエントリーしない）のか、実行エラーなのかをログで確認する。

6. **次回比較の固定指標**
   次回 run との比較は avg_net_profit / avg_profit_factor / avg_max_drawdown / MSTR 集中度(%) / tp-q パラメータ感度の 5 軸で行う。
