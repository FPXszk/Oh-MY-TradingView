# Dr.K チャート投資戦略の定量化・自動判定 調査レポート

## 結論

**実現性はかなり高いです。**

Dr.K氏の判断は一見「チャートを見た感覚」に見えますが、投稿を分解すると、実際には次の順番で判断しています。

> **地合い → 強いセクター → リーダー株 → 相場の段階 → セットアップの締まり → ブレイク位置 → 損切り位置**

したがって、単純な画像認識AIよりも、

1. OHLCVから形を数値化
2. 銘柄を「現在の状態」に分類
3. 状態ごとにスコアリング
4. 最後にAI画像判定を補助利用

という構成が適しています。

今回、公開検索で確認できた**2024年12月～2026年7月の投稿**を中心に、チャート・エントリー・押し目・損切り・地合いに関する発言を重複統合しました。ただし、Xの公開検索に表示されない投稿もあるため、「全投稿を完全取得したアーカイブ」ではありません。完全網羅にはX APIや本人の投稿エクスポートが必要です。

# 1. Dr.K氏のチャート戦略を言語化した全体像

## 相場初期：持ち合いブレイクで初玉を作る

最低でも数か月続いたレンジでエネルギーが蓄積され、突然大幅な出来高増加を伴って抜ける銘柄を探しています。「10倍以上の出来高」は、通常のブレイクすべての必須条件というより、特に大化け候補を見つけるための強条件と考えるのが自然です。([x.com](https://x.com/Drdebuneko/status/2051171801158586698))

本人が買うトリガーとして明示しているのは、主に次の2つです。

- 明確な価格節目のブレイク
- トレンドラインのブレイク

イビデンの例では、同じ上昇トレンド内で複数回入り直し、10日線割れで部分利確する運用も説明しています。([x.com](https://x.com/Drdebuneko/status/2068321904189169886))

## 相場中盤：高値のN字・U字で回転する

理想形は、下の持ち合いブレイクで取得した初玉をできるだけ保有しつつ、上昇中盤以降に高値圏で形成されるN字・U字を使って追加・回転する形です。([x.com](https://x.com/Drdebuneko/status/2073384226502943011))

ブレイクを見逃した場合も、そのまま飛びつくのではなく、

- 5日線・10日線が追いつく
- 小さなN字・U字が形成される
- その新しい小型セットアップを抜く

という順番を待つことを勧めています。([x.com](https://x.com/Drdebuneko/status/2052896276904387065))

## 高値で一度止められても、すぐには捨てない

N字で前回高値を一度で超えられなくても、高値付近で横ばいを続けている場合は、売り圧力を吸収している可能性があり、その後に上昇することがあると説明しています。([x.com](https://x.com/Drdebuneko/status/2067142178728714399))

つまり、単なる「高値を抜けなかった失敗」ではなく、

- 高値から大きく離れない
- 安値を切り下げない
- 値幅が縮小する
- 高値付近に滞在する

なら、**吸収型セットアップ**として残す考え方です。

## ブレイクから離れすぎた場所では買わない

本人が数値としてかなり明確に述べているのが、

> ブレイク地点から5％以上上で買うと、損切りになる確率が高い

という経験則です。([x.com](https://x.com/Drdebuneko/status/2067810666560508044))

ただし、これは「最初のブレイク地点から永遠に5％以内」という意味ではありません。

株価が上昇したあとに新しいN字・U字・三角持ち合いを作った場合は、**その新しいパターンの上限が次のピボット**になります。したがってプログラムでは、常に「最新の有効なセットアップのピボット」からの乖離率を計算する必要があります。

## 移動平均線の役割

投稿から読み取れる役割分担は次のとおりです。

| 移動平均線 | 主な意味 |
|---|---|
| 5日線 | 非常に強いモメンタムの維持、短期の押し目 |
| 10日線 | 回転玉の管理、部分利確や入り直し判断 |
| 25日線 | 日本株の上昇トレンド継続を測る中期基準 |
| 週足・月足 | 上値抵抗、相場の若さ、大きな値幅余地の確認 |

強い銘柄は5日線からほとんど離れず上昇することがあり、5日線への沿い方そのものを強さの判断材料にしています。([x.com](https://x.com/Drdebuneko/status/1871793350145392999))

一方、長く5日線や10日線を割らずに上昇してきた銘柄が25日線まで下がる場合は、それまでと同じ短期トレンドではなく、変化が発生した可能性を考える必要があります。([x.com](https://x.com/Drdebuneko/status/2068118843999010853))

2倍・3倍の大きな上昇を狙う場合は、日足だけでなく、週足・月足のセットアップと過去の抵抗価格帯を見る必要があるとも述べています。([x.com](https://x.com/Drdebuneko/status/2058530729546723821))

# 2. 投稿から抽出できた機械化可能なルール

| 本人の判断 | プログラム上の解釈 |
|---|---|
| 数か月のレンジから出来高急増で抜ける | 長期ベース＋出来高ブレイク |
| 節目かトレンドライン抜けしか買わない | 明確なピボットが存在しない銘柄を除外 |
| 三角持ち合い抜けが買い位置 | 高値切り下げ＋安値切り上げの収縮検出 |
| ブレイク地点から5％以上を追わない | `extension_pct > 5%`なら元のセットアップでは新規禁止 |
| 5日線・10日線を引き付ける | 株価と短期MAの乖離縮小を待つ |
| 小さなN字・U字を抜いて買う | 押し目後の新規ピボットブレイク |
| 高値付近の横ばいを捨てない | 高値滞在＋値幅縮小＝売り圧吸収 |
| 5日線に沿う株は強い | MA5上の滞在率・MA5割れ回数を評価 |
| 10日線割れで部分利確 | 短期回転玉のトレーリング条件 |
| 25日線は上昇継続の目安 | 中期トレンド判定 |
| 地合いが悪いとブレイク成功率が落ちる | 市場レジームを事前ゲートにする |
| 所属セクター全体が上昇しているかを見る | セクター騰落率・ブレッドス・相対強度 |
| リーダー株の崩れを危険信号にする | セクター上位株のセットアップ崩壊率 |
| 上昇トレンド後期は失敗が増える | 相場段階に応じてブレイクスコアを減点 |
| 甘く緩いセットアップを買わない | ベース幅・ATR・下ヒゲ・ギャップの乱れを減点 |
| セットアップがなければ触らない | 判定不能を無理にランキングしない |

地合いと所属セクターを最重要条件に挙げており、地合いが悪いとブレイクアウト確率が著しく低下すると説明しています。([x.com](https://x.com/Drdebuneko/status/2034841303566229988))

また、上昇トレンド後期では、見た目上セットアップが整っていてもブレイク失敗が増えると述べています。([x.com](https://x.com/Drdebuneko/status/2072640877621834063))

リーダー株のセットアップが崩れた局面では、フルポジションを避け、ポジションを軽くする姿勢も一貫しています。([x.com](https://x.com/Drdebuneko/status/2070476815672746449))

# 3. 一番重要な設計：点数より先に「状態」を判定する

この戦略を一つの0～100点だけで表すと、かなり情報が失われます。

まず銘柄を以下の状態に分類すべきです。

| 状態 | 意味 | システム表示 |
|---|---|---|
| `BASE_FORMING` | 持ち合い形成中 | 監視 |
| `BREAKOUT_READY` | ピボット直下で締まっている | 今すぐ触れる候補 |
| `BREAKOUT_TRIGGERED` | 当日または直近でブレイク | エントリー候補 |
| `EXTENDED` | ピボットから離れすぎ | 飛びつき禁止 |
| `PULLBACK_FORMING` | ブレイク後の押し目形成中 | 待機 |
| `NU_READY` | 小さなN字・U字が完成直前 | 再エントリー候補 |
| `ABSORPTION` | 高値付近で売り圧吸収中 | 継続監視 |
| `TREND_HOLD` | 初玉を握る状態 | 保有継続 |
| `FAILED_BREAKOUT` | ピボットを抜いた後に失速 | 撤退候補 |
| `SETUP_BROKEN` | 支持線・MA・直近安値割れ | 損切り |
| `NO_TRADE` | 緩い・地合い不良・位置不良 | 見送り |

ユーザーが求めている分類は、この状態モデルでほぼ表現できます。

- **今すぐ触れるもの**  
  `BREAKOUT_READY` / `BREAKOUT_TRIGGERED`

- **ブレイク後に緩い押し目が来ているもの**  
  `PULLBACK_FORMING`

- **上抜けを見逃したが後から入りそうなもの**  
  `NU_READY` / `ABSORPTION`

- **すでに上がりすぎているもの**  
  `EXTENDED`
