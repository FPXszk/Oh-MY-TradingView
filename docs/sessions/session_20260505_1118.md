# Session Log 20260505_1118

## 作業概要

バックテスト結果まとめの評価軸を、composite score 中心から baseline-first に変更する方針を整理した。
ユーザーの指摘に基づき、今後の `docs/research/TEMPLATE.md` は baseline に対して各派生戦略がどれだけ利益に追随し、PF・DD対利益比率・勝率を改善したかを見る形にした。

## 背景

Night Batch Self Hosted run 94 の結果では、`ema-macd-rsi-sl-baseline` が raw profit で圧倒的に大きかった。
一方で従来テンプレートは composite score を使い、固定金額の DD や銘柄集中度も強く見ていたため、baseline の raw profit の強さが直感的に伝わりにくかった。

今回の判断:
- composite score は標準評価軸から外す。
- baseline が存在する campaign では、baseline を比較の大前提として扱う。
- 派生戦略は baseline に対する利益追随率、PF差分、DD対利益比率、勝率差分で見る。
- DD は固定金額ではなく `avg_max_drawdown / avg_net_profit` を使う。
- 銘柄集中度は標準セクションから外す。

## 変更内容

### `docs/research/TEMPLATE.md`

以下の構造へ変更した。

- ヘッダー
- 結論
- 判断基準
- Baseline 結果（比較の大前提）
- 全戦略一覧（baseline 比較）
- Top 戦略
- 除外候補
- 今回の振り返り
- 次回バックテスト確認事項

判断基準:
- `avg_profit_factor`: 従来どおり `>= 2.0` を優秀、`< 1.5` を即除外。
- `dd_to_profit`: `avg_max_drawdown / avg_net_profit`。固定DD金額ではなく利益に対するDD比率を見る。
- `avg_win_rate`: `>= 40%` 優秀、`35〜40%` 許容、`30〜35%` 要注意、`< 30%` 即除外。
- `profit_follow_rate`: `avg_net_profit / baseline avg_net_profit`。baseline への追随率を見る。

## 今後の方針

今後のバックテスト結果まとめでは、baseline を弱いものとして扱わない。
raw profit が強い baseline は利益最大化候補として保持し、派生戦略は「利益をどれだけ犠牲にして PF や DD 対利益比率を改善できたか」で評価する。

利益最大化と risk/reward 改善は別軸として結論を書く。
