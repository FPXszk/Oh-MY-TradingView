## 実装内容

### Phase 1（TradingView Scanner API のみ）

**新規ファイル:**
- `src/core/fundamental-screener.js` — スクリーナーコア
- `scripts/screener/run-fundamental-screening.mjs` — GHA エントリポイント
- `.github/workflows/daily-screener.yml` — schedule cron UTC 03:00（JST 12:00）
- `docs/reports/screener/.gitkeep` — レポート出力先

**修正ファイル:**
- `src/cli/commands/screener.js` — `fundamental` サブコマンド追加（`--with-yahoo` フラグ含む）
- `src/tools/screener.js` — `market_fundamental_screener` MCP ツール追加
- `src/core/index.js` — `runFundamentalScreener` エクスポート追加

**フィルター条件（サーバーサイド）:**
- EPS(TTM) > 0
- ROE > 15%
- 粗利率(TTM) > 40%
- FCFマージン(TTM) > 10%
- SMA200 下回り除外
- 時価総額 > $1B
- 相対出来高 > 1.2x

**フィルター条件（クライアントサイド）:**
- RSI(14) > 60
- Close > SMA50
- Close ≥ 52週高値 × 75%
- Perf.3M > 10%
- P/FCF = market_cap / FCF_TTM < 50（クライアント計算）

**スコアリング:**
`rank(Perf.3M) + rank(ROE) + rank(FCFマージン)`（合計が小さいほど上位）

### Phase 2（Yahoo Finance 売上成長率補完）

**実装内容:**
- `batchFetchRevenueGrowth()`: 5件バッチ × 500ms遅延でYahoo Financeから取得
- `enrichWithYahoo` フラグで起動
- revenueGrowth > 20% フィルター（null は通過）
- ランクスコアに `rank(revenueGrowth)` を追加

**運用状況:**
Yahoo Finance v10 API が GHA 環境で 429/401 を返すため、全銘柄 N/A。
GHA スクリプトは Phase 1 のみで運用（`enrichWithYahoo: false`）。
`--with-yahoo` フラグは実装済みで、将来 Yahoo Finance が利用可能な環境で活用可能。
